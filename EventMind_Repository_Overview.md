# Обзор репозитория: Платформа EventMind

Этот документ представляет собой высокоуровневый обзор архитектуры и кодовой базы платформы **EventMind**. Вы можете использовать его для онбординга новых разработчиков, передачи проекта коллегам или демонстрации структуры системы.

## О проекте
**EventMind** — это интеллектуальная платформа для поиска, рекомендаций и управления мероприятиями. Проект выделяется гибридной архитектурой: он объединяет мощную ERP-систему (Odoo) для управления "сырыми" данными событий, кастомный Backend на Python для сложной логики (Семанитческий поиск, Умные Рекомендации) и современный Frontend на React. Также проект глубоко интегрирован с LLM (искусственным интеллектом) для общения с пользователем.

## 🧠 Ключевые технологические "Фичи" (Features)

Ниже описан функционал, ради которого была построена данная архитектура:

### 1. Гибридные Умные Рекомендации (Recommendation Engine)
Собственный модуль контентной фильтрации (файл _Recommender_).
- Как только пользователь кликает карточки, просматривает их или добавляет в Избранное — система записывает это в кэш-базу `UserInteraction`.
- Оценка (Score) формируется из: соответствия интересов профиля тегам мероприятия, наличия свободного времени в календаре клиента (hard filter) и буста от недавних взаимодействий с похожими темами.
- **Интеграция с ИИ:** Нейросеть просматривает ТОП-3 финальных рекомендаций и на лету пишет текст (Personal AI Advice) на тему того, *почему конкретно* человеку стоит туда пойти.

### 2. Умный поиск (GigaChat AI Search)
В платформу внедрена LLM (Сбер GigaChat). Строка поиска понимает свободные запросы. Можно спросить _"Найди лекции по дизайну на этой неделе либо вечером, либо на выходных"_ — система отправит промпт в Python-бекэнд, LLM распарсит свободный запрос, сопоставит его с доступной базой из Odoo и вернет готовый текстовый ответ на фронтенд в `AiResponseModal.jsx`. 

### 3. Интеграция с Odoo ERP
Backend не хранит события жестко внутри себя, выступая прокси-мостом. Он связывается с мощной `odoo_db` через XML-RPC протокол. Для ускорения тяжелых запросов в API-Gateway внедрено In-Memory кэширование объектов.

### 4. Календарь и Расписание (Scheduler)
Модуль расписания `schedule` позволяет добавлять личные события. Запускается фоновый планировщик (`BackgroundScheduler` / `apscheduler`), который может парсить `UserFavorite` и базу расписаний для отправки нотификаций/напоминаний о скором начале событий.
